# Standards

This directory contains organization-wide engineering standards that apply to all projects regardless of language, framework, or domain.

## Contents

| File | Purpose |
|------|---------|
| `coding-standards.md` | Universal coding principles that apply across languages |
| `documentation-standards.md` | Documentation requirements for code, APIs, and projects |
| `testing-standards.md` | Test strategy, coverage requirements, and quality standards |
| `git-workflow.md` | Branching strategy, commit conventions, and PR standards |
| `api-standards.md` | REST API design conventions and versioning strategy |

## Priority

These standards are the **baseline**. Language-specific standards in `skills/languages/` and framework-specific standards in `skills/frameworks/` extend and specialize these baselines. When in conflict, the more specific standard (language/framework) takes precedence.
